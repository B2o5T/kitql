Once everything is setup, just use `KitQL`!

## Queries

### In a route

If you do this in the route, you will have the data in `SSR` mode.

```svelte
<script context="module" lang="ts">
	import { AllContinentsQuery	} from '$lib/graphql/_kitql/graphqlStores';

	export async function load({ fetch }) {
    // Will fill `$AllContinentsQueryStore`
		await AllContinentsQuery({ fetch });
		return {};
	}
</script>

<!-- Just use the store here, or anywhere else! -->
{JSON.stringify($AllContinentsQueryStore.data, null, 2)}
```

### In a component

```svelte
<script lang="ts">
  import { browser } from '$app/env';
  import { AllContinentsQuery, AllContinentsQueryStore } from '$lib/graphql/_kitql/graphqlStores';

  $: browser && AllContinentsQuery();
</script>

<!-- Just use the store here, or anywhere else! -->
{JSON.stringify($AllContinentsQueryStore.data, null, 2)}
```

### Query Options

```svelte
<script lang="ts">
  import { AllContinentsQuery, AllContinentsQueryStore } from '$lib/graphql/_kitql/graphqlStores';

  async function cacheAndNetwork() {
    await AllContinentsQuery({ settings: { policy: 'cache-and-network' } });
  }
</script>

<!-- Just use the store here, or anywhere else! -->
{JSON.stringify($AllContinentsQueryStore.data, null, 2)}
```

### Reset Cache

```svelte
<script lang="ts">
  import { AllContinentsQueryCacheReset, AllContinentsQueryStore } from '$lib/graphql/_kitql/graphqlStores';

  function cacheReset() {
    AllContinentsQueryCacheReset();
  }
</script>

<!-- Just use the store here, or anywhere else! -->
{JSON.stringify($AllContinentsQueryStore.data, null, 2)}
```

### Store Update

```svelte
<script lang="ts">
  import { AllContinentsQueryStoreUpdate, AllContinentsQueryStore } from '$lib/graphql/_kitql/graphqlStores';

  function cacheReset() {
    AllContinentsQueryStoreUpdate([{ name: 'JYC Land', code: 'JYC' }], 'continents');
  }
</script>

<!-- Just use the store here, or anywhere else! -->
{JSON.stringify($AllContinentsQueryStore.data, null, 2)}
```
